 This is my first javascript project. When I first tried to learn javascript, I was focused on only the theories of javascript. Studying by taking notes, and memorizing javascript concepts only showed me how complex javascript is rather than showing me the purpose of using this language. But I could not give up as I thought javascript was a step that I must take in order to move forward towards my future career. I decided to try project-based learning and searched for javascript projects online. I clone coded this simple tetris game by typing each code and understanding it as I went through. To test myself if I fully understand my code, I added more features to the game such as pause, hold block, and show next block. Adding hold block function was easy as all I had to do was erase current moving block and re-rendering it to hold box when a button was pressed. However, adding a function to show the next block required me to go through my code many times. I had to make changes to functions that initialize the game to start so that the generated blocks renders in queue box first instead of rendering immediately on the screen. In order to implement this function, I used two variables, tempNextItem and nextItem, to save the generated blocks in the queue box before the current moving block is seized. From this project, I learned the importance of project-based learning and going further than just cloning the codes to fully understand what I programmed.

 After getting familiar with javascript, I went on to learn a very useful and popular library: React.js
By using create-react-app, I learned about states, props, JSX, fetching data from api service, and routing to different pages. As it was my first time using api keys to fetch data, I did not know that I must hide my api key from my user. I learned about .env files and .gitignore files after searching for methods to hide my api key before pushing my project to github. However, my api key was still shown in network requests of my site. To fix this, I re-created my project on a framework called next.js. Next.js provided url re-write function so that I can first send the user to a temporary page before fetching the data, and then re-direct the user to actual page after masking the url. This way, the user is not able to view requests and does not know the difference in the url.
